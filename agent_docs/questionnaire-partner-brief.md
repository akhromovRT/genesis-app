## 1. Что это и зачем

Genesis — платформа персонального здоровья и долголетия. Модуль **биохакинг-анкеты** — это первая точка контакта пользователя с системой (слой «Onboarding & Questionnaire» в архитектуре Genesis).

**Задачи модуля:**
- Собрать структурированный анамнез клиента (образ жизни, питание, здоровье, лекарства, симптомы)
- Автоматически выявить зоны риска и возможные дефициты
- Работать как бесплатная лид-воронка: анкету можно пройти без регистрации
- Готовить контекст для будущего Genesis Coach (AI-ассистент) и Protocol Engine (AI-протокол)

**Источник:** две профессиональные анкеты от нутрициолога (docx):
- Часть 1: Образ жизни, Лекарства, БАД
- Часть 2: Питание и Нутриенты

## 2. Как это выглядит для пользователя

### Точки входа на сайте
- Кнопка **«Пройти анкету»** в главном меню (header) и footer
- Кнопка **«Пройти анкету»** в hero-блоке на главной странице
- Кнопка **«Пройти бесплатную анкету»** в финальном CTA на главной
- В dashboard авторизованного пользователя: пункт **«Моя анкета»** в боковом меню

### Пользовательский путь (анонимный)

```
1. genesisbio.ru/questionnaire → Лендинг-страница анкеты
   - Объяснение: 9 шагов, ~25 минут, 15 подсветок
   - Кнопка «Начать анкету»

2. /questionnaire/start → Визард (многошаговая форма)
   - Шаг 0: Введение (что получите, дисклеймер, GDPR)
   - Шаг 1: О себе (пол, возраст, рост, вес, ОТ/ОБ)
   - Шаг 2: Образ жизни (работа, активность, сон, режим дня, вода)
   - Шаг 3: Анамнез (семейные болезни, хронические, операции)
   - Шаг 4: Системы организма (10 систем: мозг, зрение, ЛОР, ССС, ЖКТ, мочеполовая, кожа, щитовидка, лёгкие, суставы)
   - Шаг 5: Стресс и аллергии (уровень 1-5, поллиноз, пищевые, лекарственные)
   - Шаг 6: Лекарства (23 группы × 4 частоты) + БАД (9 категорий × 4 частоты)
   - Шаг 7: Питание (17 категорий продуктов, напитки, пищевые предпочтения)
   - Шаг 8: Симптомы дефицитов (30 состояний) + цели + согласие

   Прогресс-бар вверху: «Шаг 4 из 8 • ~16 мин осталось»
   Автосохранение в браузере после каждого шага
   Можно выйти и вернуться — прогресс сохранён

3. /questionnaire/result/[token] → Страница результатов
   - Блок подсветок: цветные карточки (красные — warning, оранжевые — attention, синие — info)
   - Каждая подсветка раскрывается: что сработало, рекомендация, дисклеймер
   - Аккордеон с ответами по всем разделам
   - CTA: «Зарегистрируйтесь, чтобы сохранить анкету»

4. /register?sessionToken=XXX → Регистрация
   - При регистрации анонимная анкета автоматически привязывается к аккаунту
   - Пользователь попадает в /dashboard/questionnaire — видит свою анкету
```

### Пользовательский путь (авторизованный)

```
1. Авторизованный пользователь проходит тот же визард
2. Прогресс сохраняется на сервере (не только в браузере)
3. После завершения — результат доступен в /dashboard/questionnaire
4. Нет CTA на регистрацию (уже зарегистрирован)
```

## 3. Движок подсветок — 15 правил

Ключевая ценность анкеты — **автоматический анализ ответов** по 15 детерминированным правилам. Это не AI/LLM, а чёткие if/then-правила, полностью тестируемые и юридически защищённые.

### Правила по категориям

| # | Категория | Правило | Когда срабатывает | Severity |
|---|---|---|---|---|
| 1 | Нутриенты | Дефицит магния | 3+ из: судороги, тремор, плохой сон, раздражительность | attention |
| 2 | Нутриенты | Дефицит B12/фолатов | 3+ из: утомляемость, одышка, бледность, жжение языка, brain fog | attention |
| 3 | Нутриенты | Дефицит железа | 3+ из: бледность, утомляемость, одышка, выпадение волос, пика | attention |
| 4 | Нутриенты | Дефицит витамина D | 2+ из: боль в костях, слабость мышц, частые простуды, ночное зрение | attention |
| 5 | Нутриенты | Дефицит Омега-3 | 2+ симптома сухости + редко жирная рыба | attention |
| 6 | Нутриенты | Дефицит цинка | 2+ из: потеря вкуса, медленное заживление, простуды, выпадение волос | attention |
| 7 | Нутриенты | Мало клетчатки | Овощей <250г + редко крупы + упомянуты запоры (2+) | info |
| 8 | Нутриенты | Избыток сахара | Часто сладкое + часто мучное + сильная тяга (2+) | attention |
| 9 | Лекарства | Высокая фарм-нагрузка | 5+ групп препаратов «часто» | **warning** |
| 10 | Лекарства | Антибиотики + ЖКТ | Антибиотики иногда/часто + жалобы ЖКТ | attention |
| 11 | Образ жизни | Мало воды | Менее 1л/день + кофе 3+ чашки | info |
| 12 | Образ жизни | Нарушены циркадные ритмы | Последняя еда <3ч до сна + плохой сон | attention |
| 13 | Образ жизни | Низкая активность | physicalActivity = low | attention |
| 14 | Стресс | Высокий стресс | Уровень 4-5 + нарушения сна/раздражительность | **warning** |
| 15 | Образ жизни | Регулярный алкоголь | 3+ раз/неделю (regex по тексту) | attention |

**Каждая подсветка содержит:**
- Название и описание на русском
- Список «что сработало» (прозрачность для пользователя)
- Конкретная рекомендация (что проверить / к какому врачу)
- Обязательный дисклеймер: «Не является медицинским диагнозом»

**Технически:** правила вычисляются на сервере (не в браузере), тестируются через Vitest (12 unit-тестов), результаты сохраняются в БД как JSONB.

## 4. Техническая архитектура

### Стек

| Компонент | Технология |
|---|---|
| Frontend | Next.js 16, React 19, TypeScript 5, Tailwind CSS 4, shadcn/ui |
| Формы | react-hook-form 7 + Zod v4 (валидация) |
| Backend | Next.js API Routes |
| БД | PostgreSQL 17 (self-hosted), Drizzle ORM |
| Auth | Custom JWT (jose + bcryptjs + HttpOnly cookies) |
| Тестирование | Vitest 4 (первый тестовый фреймворк в проекте) |
| Деплой | Docker → GitHub Actions → VPS → Nginx → genesisbio.ru |

### База данных

Одна таблица `questionnaire_sessions`:

| Поле | Тип | Описание |
|---|---|---|
| id | UUID | PK |
| user_id | UUID nullable | null = анонимный, FK → profiles |
| session_token | TEXT unique | Токен для LocalStorage (21 символ) |
| status | TEXT | 'draft' или 'completed' |
| current_step | INT | 0..9 |
| answers | JSONB | Все ответы, структурированные по шагам |
| highlights | JSONB | Массив сработавших правил (вычисляется серверно) |
| completed_at | TIMESTAMPTZ | Дата завершения |
| email | TEXT nullable | Email из soft-gate |
| created_at / updated_at | TIMESTAMPTZ | Автоматические метки |

### Структура файлов

```
src/lib/questionnaire/           ← Ядро бизнес-логики
  types.ts                       ← Zod-схемы всех 9 шагов (~200 строк)
  steps-config.ts                ← Метаданные шагов (title, subtitle, время)
  storage.ts                     ← LocalStorage обёртка
  rules.ts                       ← 15 правил подсветок (~400 строк)
  highlights.ts                  ← computeHighlights() — движок
  *.test.ts                      ← 12 unit-тестов

src/app/api/questionnaire/       ← API (3 endpoint'а)
  submit/route.ts                ← POST: завершение + вычисление highlights
  session/route.ts               ← GET: загрузка черновика / PATCH: обновление
  attach/route.ts                ← POST: привязка анон-сессии к userId

src/components/questionnaire/    ← UI (~16 компонентов)
  wizard.tsx                     ← Контейнер с навигацией и persistence
  steps/step-0..step-8.tsx       ← 9 шаговых компонентов
  shared/frequency-table.tsx     ← Таблица «Никогда/Редко/Иногда/Часто»
  shared/symptom-checklist.tsx   ← Чеклист 30 симптомов
  shared/progress-bar.tsx        ← Прогресс-бар сверху
  summary/highlight-card.tsx     ← Одна карточка подсветки
  summary/highlights-panel.tsx   ← Панель подсветок
  summary/summary-view.tsx       ← Полная сводка

src/app/(public)/questionnaire/  ← Страницы
  page.tsx                       ← Лендинг
  start/page.tsx                 ← Визард
  result/[sessionToken]/page.tsx ← Публичная сводка
src/app/dashboard/questionnaire/
  page.tsx                       ← Авторизованная сводка
```

### API endpoints

| Метод | Endpoint | Назначение | Auth |
|---|---|---|---|
| POST | `/api/questionnaire/submit` | Завершить анкету, вычислить highlights | Опционально |
| GET | `/api/questionnaire/session?token=X` | Загрузить черновик | Если session.userId — проверка |
| PATCH | `/api/questionnaire/session` | Обновить черновик | Если session.userId — проверка |
| POST | `/api/questionnaire/attach` | Привязать анон к user | Обязательно |

## 7. Дальнейшие шаги

### Ближайшие (неделя)

1. **Тестирование с реальными пользователями** — попросить 3-5 человек пройти анкету, собрать UX-фидбек: понятность шагов, длительность, ценность подсветок
2. **Email-уведомление** — при завершении анкеты отправлять сводку на email (когда настроим Resend)
3. **Красивая сводка** — заменить JSON-отображение ответов на человекочитаемый формат по всем шагам

### Среднесрочные (месяц)

4. **Интеграция с анализами крови** — результаты анкеты + биомаркеры = перекрёстные подсветки (например, дефицит B12 в анкете + низкий B12 в крови = подтверждённый дефицит)
5. **Расширение правил** — добавить 10-15 новых правил на основе перекрёстных данных (генетика + анкета + кровь)
6. **Валидированные шкалы** — добавить стандартизованные опросники (PHQ-9 для депрессии, PSS для стресса, PSQI для сна) как отдельные «мини-анкеты»
7. **Повторное прохождение** — сравнение «было/стало» через 3-6 месяцев

### Стратегические (квартал)

8. **Genesis Coach** — AI-ассистент, который использует анкету как контекст для персонализации: «Вы указали мышечные судороги и плохой сон — проверим магний в крови»
9. **Protocol Engine** — персональный протокол на основе всех данных (анкета + генетика + кровь + микробиом)
10. **Корпоративный модуль** — массовое анкетирование сотрудников → агрегированный отчёт для HR (без персональных данных)

### Технический долг

11. **Красивое форматирование сводки** — step2-step8 сейчас в JSON, нужен человекочитаемый вывод
12. **Rate limiting на submit** — защита от спама (5 запросов/минуту/IP). Пока не реализовано.
13. **Расширенные тесты** — E2E через Playwright (прохождение анкеты от начала до конца)
14. **Миграция на Docker** — PostgreSQL 17 сейчас установлен через brew локально; на VPS используется Docker-контейнер

## 8. Как запустить локально

```bash
# 1. Клонировать и установить зависимости
git clone git@github.com:akhromovRT/genesis-app.git
cd genesis-app
npm install

# 2. PostgreSQL (уже настроен на MacBook Алексея)
brew services start postgresql@17
# БД genesis, пользователь pass24 — уже созданы

# 3. Накатить миграции (если БД пустая)
psql postgresql://pass24:8134cb20109a053c55369ac6e232f1d0@localhost:5432/genesis \
  -f drizzle/0001_init.sql \
  -f drizzle/0002_dna_tables.sql \
  -f drizzle/0003_questionnaire.sql

# 4. Запустить dev-сервер
npm run dev
# → http://localhost:3000/questionnaire

# 5. Запустить тесты
npm test
```

## 9. Ключевые решения (почему так, а не иначе)

| Решение | Альтернатива | Почему выбрали |
|---|---|---|
| JSONB-blob для ответов | Нормализованные таблицы | Анкета будет эволюционировать; JSONB гибче для MVP, нормализуем когда стабилизируется |
| 15 детерминированных правил | LLM-интерпретация | Правила тестируемы, предсказуемы, дёшевы, юридически защищены дисклеймерами |
| LocalStorage для анонимов | Серверные сессии | Не нужна БД для черновиков, прогресс хранится у пользователя, нет затрат на хранение |
| Soft-gate (показываем сводку, но CTA на регистрацию) | Hard-gate (сводка только после регистрации) | Не отпугиваем пользователя; показываем ценность, потом конвертируем |
| Vitest (не Jest) | Jest | Vitest нативно поддерживает ESM, TypeScript, path aliases без конфигов; быстрее Jest |
| Серверное вычисление highlights | Клиентское | Один источник правды; пользователь не может подделать результат |
