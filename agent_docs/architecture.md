# Архитектура

## Обзор

Genesis — персональная операционная система здоровья и долголетия. Три ключевых слоя:

1. **Genetic Foundation** — анализ ДНК по 7 доменам (метаболизм, воспаление, детоксикация, долголетие, сон, спортивный потенциал, усвоение нутриентов). Источники: 23andMe / AncestryDNA raw data, партнёрский тест.
2. **Dynamic Biomarker Layer** — импорт и longitudinal-трекинг анализов крови (60+ маркеров), микробиома, данных носимых устройств. Выдаёт биологический возраст, тренды и контекстные алёрты.
3. **Personalized Protocol Engine + Genesis Coach** — AI генерирует персональный протокол на пересечении генетики, биомаркеров и образа жизни. Встроенный AI-ассистент отвечает на вопросы по данным пользователя, формирует ежемесячные health-отчёты.

## Контекст

- Стадия: концепция / pre-seed (март 2026). Кода ещё нет.
- Целевая аудитория (B2C): биохакеры, предприниматели, топ-менеджеры, спортсмены-любители, 30–55 лет.
- B2B2C-вектор: корпоративные health-программы, превентивные клиники, страховые компании.
- Конкуренты: InsideTracker (кровь), 23andMe (генетика), Viome (микробиом), WHOOP/Oura (wearables). Ни один не объединяет все слои.

## Ключевые компоненты

| Компонент | Назначение |
|---|---|
| Genetic Foundation | Парсинг и интерпретация raw DNA-данных, расчёт предрасположенностей |
| Biomarker Importer | Импорт результатов анализов крови (ручной ввод / PDF / интеграции лабораторий) |
| Microbiome Module | Импорт и интерпретация данных микробиома |
| Wearables Connector | Интеграция с носимыми устройствами (Apple Health, Oura, WHOOP и др.) |
| Bio Age Engine | Расчёт биологического возраста и трендов на основе всех слоёв данных |
| Protocol Engine | AI-генерация персонального протокола (питание, нутрицевтики, образ жизни) |
| Genesis Coach | AI-ассистент: Q&A по данным пользователя, ежемесячные отчёты, адаптация протокола |
| Ретест-контур | Отслеживание эффекта вмешательств в динамике |

## Потоки данных

```
Источники данных:
  23andMe / AncestryDNA (raw DNA file)
  Лаборатории (PDF / API / ручной ввод) → 60+ биомаркеров
  Микробиомные тесты (партнёрские наборы)
  Wearables (Apple Health, Oura, WHOOP и др.)
        │
        ▼
  Единое хранилище данных пользователя (longitudinal)
        │
        ▼
  AI-движок (Protocol Engine)
        │
        ├─→ Персональный протокол (питание, нутрицевтики, lifestyle)
        ├─→ Биологический возраст + тренды
        ├─→ Контекстные алёрты
        └─→ Genesis Coach (AI-чат + monthly reports)
```

## Технологии и зависимости

Технологический стек не определён (стадия концепции). Определять в процессе PRD и технического проектирования.

Предполагаемые требования к стеку:
- Мобильное приложение (iOS приоритет, Android — второй этап)
- Бэкенд с поддержкой longitudinal-хранения данных
- AI/LLM-интеграция для Protocol Engine и Genesis Coach
- Безопасное хранение чувствительных медицинских данных (HIPAA / GDPR-совместимость)

## Нефункциональные требования и ограничения

- **Приватность и безопасность:** генетические и медицинские данные — максимально чувствительная информация. Требования: шифрование, минимизация данных, compliance (HIPAA / GDPR).
- **Точность рекомендаций:** протокол должен иметь методологическую базу (партнёрство с экспертами в биохакинге/медицине).
- **Longitudinal-хранение:** вся история данных пользователя хранится для отслеживания эффекта вмешательств.
