# Архитектура

## Обзор

Genesis — персональная операционная система здоровья и долголетия. Четыре ключевых слоя:

0. **Onboarding & Questionnaire** — точка входа: знакомство с концепцией, стартовое анкетирование (анамнез), сбор ранее выполненных анализов и меддокументов. Три воронки входа: генетика (база), кровь (холодный трафик), микробиом (ЖКТ-аудитория).
1. **Genetic Foundation** — анализ ДНК по 7 доменам (метаболизм, воспаление, детоксикация, долголетие, сон, спортивный потенциал, усвоение нутриентов). Включает блок фармакогенетики (маршрутизация лекарств). Источники: 23andMe / AncestryDNA raw data, партнёрский тест, авторская ДНК-панель.
2. **Dynamic Biomarker Layer** — импорт и longitudinal-трекинг анализов крови (60+ маркеров), микробиома (МСММ/ХМС — предпочтительный метод), данных носимых устройств. Лонжевити-оптимальные диапазоны вместо стандартных референсов. Выдаёт биологический возраст (PhenoAge и др.), тренды и контекстные алёрты.
3. **Personalized Protocol Engine + Genesis Coach** — AI генерирует персональный протокол по «пирамиде»: данные → биоритмы → питание/нутрицевтика → антистресс. Две линии: снижение симптомов + профилактика рисков. «Умное хранилище» с подсветкой рисков и разделением генетических/поведенческих факторов. Чатовый AI-ассистент, ежемесячные health-отчёты. Обязательные юридические дисклеймеры.

## Контекст

- Стадия: концепция / pre-seed (март 2026). Кода ещё нет.
- Целевая аудитория (B2C): биохакеры, предприниматели, топ-менеджеры, спортсмены-любители, 30–55 лет.
- B2B2C-вектор: корпоративные health-программы, превентивные клиники, страховые компании.
- Конкуренты: InsideTracker (кровь), 23andMe (генетика), Viome (микробиом), WHOOP/Oura (wearables). Ни один не объединяет все слои.

### Пять блоков оценки (из совещания 2026-03-23)

1. **Биологический возраст** (в т.ч. PhenoAge) — ключевой мотиватор и метрика эффективности
2. **Кровь** — нутриентная плотность, воспаление, витамины/минералы/тяжёлые металлы
3. **Микробиота** — метрика эффективности интервенций
4. **Генетика** — базовая платформа (одноразовая)
5. **Опросники** — анамнез, валидированные шкалы, биохакинг-анкета

### Longevity Pyramid — научный фреймворк Protocol Engine

> Источник: Martinović et al. (2024) "Climbing the longevity pyramid" — *Frontiers in Aging* 5:1495029

Protocol Engine генерирует рекомендации по 5 прогрессивным уровням пирамиды:

| Уровень | Название | Что реализует Genesis | Охват |
|---|---|---|---|
| 1 | Diagnostics & Analysis | Bio Age Engine, Longevity Calculator, Biomarker Importer, Genetic Foundation | MVP |
| 2 | Lifestyle Interventions | Рекомендации: упражнения (VO₂max, 150–300 мин/нед), питание (CR/IF/MedDiet), сон, стресс | MVP |
| 3 | Dietary Supplements | Персональный стек нутрицевтиков (NAD+, спермидин, AKG, фисетин, кверцетин, пробиотики) по генетике + биомаркерам | v2 |
| 4 | Pharma & Non-pharma | Информирование о процедурах (сауна, HBOT, криотерапия); фармакология (рапамицин, метформин) — только через врача | v3+ |
| 5 | Experimental | Образовательный контент (CRISPR, стволовые клетки, мРНК) без рекомендаций | контент |

Подробный фреймворк с дозировками, механизмами и ссылками: `conept/genesis_longevity_pyramid.md`

### Модуль Longevity Calculator (референс: longevity-tools.com)

Публичный (freemium) модуль калькуляторов и интерпретаторов — аналог longevity-tools.com, но интегрированный в экосистему Genesis. Работает как отдельная точка входа и как инструмент внутри приложения.

**Калькуляторы биологического возраста:**
- Levine PhenoAge — 9 рутинных маркеров крови (Albumin, Creatinine, Glucose, hsCRP, LYM%, MCV, RDW, ALP, WBC) + возраст → биовозраст + 10-летний риск смертности
- Bortz Blood Age — более продвинутая модель на коммерческих лаб-тестах

**Интерпретаторы по системам:**
- Глюкозный метаболизм (мульти-маркерная оценка контроля сахара)
- Функция печени (автоматический расчёт индексов здоровья печени)
- Статус железа (Body Iron Stores + персонализированный план)
- Планируемые: масса тела, мужские гормоны, щитовидная железа, артериальное давление, ASCVD-риск

**Расчётные индексы (18+):**
- Инсулинорезистентность: HOMA-IR, QUICKI, TyG, TyG-BMI
- Печень: FIB-3, FLI, LSS, ANI, R-Factor, D/T Bilirubin
- Щитовидная железа: SPINA-GT, SPINA-GD, TSHI, PTFQI, FT3/FT4, fT3/rT3
- Кардио: Risk-weighted ApoB, Triglyceride/HDL

**Оптимальные диапазоны биомаркеров (80+):**
- 9 категорий: Blood Sugar, Lipids, Kidney, Liver, Blood Health, Thyroid, Sex Hormones, Immunity, Tumor
- Лонжевити-оптимальные (не стандартные лабораторные) с ссылками на исследования

**UX-паттерн (по longevity-tools.com):**
- Ввод значений с поддержкой множественных единиц измерения (автоконвертация)
- Мгновенный результат без регистрации
- Подробные объяснения формул ("Details for nerds")
- Ссылки на оригинальные исследования
- Возможность поделиться результатом через URL

**Интеграция с Genesis:**
- Калькулятор как бесплатная публичная страница → воронка конверсии в полный продукт
- Внутри приложения: автоматический расчёт индексов из импортированных биомаркеров (без ручного ввода)
- Лонжевити-тренды: отслеживание изменения биовозраста и индексов во времени

## Технологии и зависимости

Технологический стек не определён (стадия концепции). Определять в процессе PRD и технического проектирования.

Предполагаемые требования к стеку:
- Мобильное приложение (iOS приоритет, Android — второй этап)
- Бэкенд с поддержкой longitudinal-хранения данных
- AI/LLM-интеграция для Protocol Engine и Genesis Coach
- Безопасное хранение чувствительных медицинских данных (HIPAA / GDPR-совместимость)

## Нефункциональные требования и ограничения

- **Приватность и безопасность:** генетические и медицинские данные — максимально чувствительная информация. Требования: шифрование, минимизация данных, compliance (HIPAA / GDPR / 152-ФЗ).
- **Юридические ограничения:** диагнозы не ставятся; интерпретации — для повышения осведомлённости и помощи врачу. Обязательные дисклеймеры на всех экранах/отчётах. Биохакинг = «медицина для здоровых людей», не лечение.
- **Доказательная база:** лонжевити-оптимальные диапазоны подкрепляются ссылками на РКИ. Уровни доказательности (А1 и др.) отражаются в отчётах. «Серые зоны» маркируются вероятностно.
- **Точность рекомендаций:** протокол должен иметь методологическую базу (партнёрство с экспертами). Консилиумный подход: анамнез + фильтры разных специалистов.
- **Longitudinal-хранение:** вся история данных пользователя хранится для отслеживания эффекта вмешательств.
- **Валидация загрузки:** алгоритм не должен «додумывать» данные; определить форматы, валидацию, обработку ошибок.
