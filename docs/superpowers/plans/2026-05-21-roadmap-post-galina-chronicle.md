# Roadmap: «product ready to promote» — пост-хроника Галины

> **Назначение:** мастер-документ, координирующий независимые планы реализации, вытекающих из ревизии хроники переписки с Галиной Хусаиновой (60 дней, 23.03 → 21.05.2026). Каждый пункт ниже — отдельный план, который ссылается на собственный файл и выполняется независимо.
>
> **Принцип:** план пишется непосредственно перед тем, как берёшь фичу в работу (а не все вперёд) — чтобы зафиксировать актуальные паттерны кода и не плодить устаревшие черновики. Этот документ обновляется по мере появления готовых планов.

> **Поворот стратегии 2026-05-21 (ADR-001):** MVP-аватар сжат до **одной женской ЦА 30+**, продуктовая линейка — **3 тарифа разовых ген-продуктов по болям** + подписка на платформу. Multi-tenant «Семья», «Профи» (consultant SaaS), «Клиника» (B2B SaaS), расширение на мужскую ЦА — **Phase 2 после релиза MVP**. Этот документ переразмечен ниже.

## Цель

После выполнения всех планов ниже Genesis имеет:
- B2C-продукт для женщины 30+ с прогрессивной воронкой (квик-результат за 1 минуту) и удержанием (валидированные опросники, челленджи, прогноз);
- Линейку из **3 тарифов разовых ген-продуктов по болям клиента** (бьюти-якорь → +стройность/гормоны → +полное wellness) + подписочный второй контур;
- Возможность загрузить любой сторонний ДНК-отчёт и получить оценку достоверности по актуальной базе;
- Кейсовую базу (10–20 обезличенных женских кейсов Галины), прошедшую через стандартный пользовательский ЛК;
- Обновлённый питч (под женскую ЦА и 3 тарифа) и контент-блок (2 якорных статьи) под промо.

### Phase 2 (после релиза MVP — отложено по ADR-001)

| № | Фича | Причина отложения |
|---|------|------------------|
| P2-1 | Multi-tenant «Семья» (до 10 профилей под подпиской) | Преждевременная сложность модели данных до доказанной конверсии на одиночном пользователе |
| P2-2 | Тариф «Профи» (нутрициологи / тренеры / врачи) | Вне MVP-аватара; требует ролевой модели и многих экранов |
| P2-3 | B2B-вертикаль «Клиника» | Долгий sales cycle; вне ёмкости pre-seed команды |
| P2-4 | Расширение на мужскую ЦА | После доказанной конверсии на женском аватаре |

## Принципы для всех планов

- **TDD по умолчанию** (Vitest уже в проекте после `2026-04-05`).
- **Малые коммиты** — после каждой задачи план содержит коммит.
- **Не плодить файлы** — следовать `src/lib/...` + `src/components/...` структуре.
- **Никаких backwards-compat shims** — проект на стадии MVP, ломать LocalStorage-форматы можно, добавив migration в `storage.ts`.
- **Документация — после задачи, не раньше:** `agent_docs/development-history.md` пополняется по факту, а не авансом.
