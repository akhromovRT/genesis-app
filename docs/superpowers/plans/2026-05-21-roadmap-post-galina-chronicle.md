# Roadmap: «product ready to promote» — пост-хроника Галины

> **Назначение:** мастер-документ, координирующий 11 независимых планов реализации, вытекающих из ревизии хроники переписки с Галиной Хусаиновой (60 дней, 23.03 → 21.05.2026). Каждый пункт ниже — отдельный план, который ссылается на собственный файл и выполняется независимо.
>
> **Принцип:** план пишется непосредственно перед тем, как берёшь фичу в работу (а не все вперёд) — чтобы зафиксировать актуальные паттерны кода и не плодить устаревшие черновики. Этот документ обновляется по мере появления готовых планов.

## Цель

После выполнения всех планов ниже Genesis имеет:
- B2C-продукт с прогрессивной воронкой (квик-результат за 1 минуту) и удержанием (валидированные опросники, челленджи, прогноз);
- Модульный каталог генетических SKU + два маркетинговых пакета (Бьюти/Веллнес);
- Возможность загрузить любой сторонний ДНК-отчёт и получить оценку достоверности по актуальной базе;
- Multi-tenant подписку «Семья» (до 10 профилей);
- Кейсовую базу (10–20 обезличенных биохакеров Галины), прошедшую через стандартный пользовательский ЛК;
- Обновлённый питч и контент-блок (2 якорных статьи) под промо.

Ветки «Профи» (consultant SaaS) и «Клиника» (B2B SaaS) сознательно вынесены за рамки и не входят в этот роадмап.

## Порядок исполнения (рекомендуемый)

**Сначала — то, что повышает конверсию или нужно к инвест-встрече (1–2 недели):**
1. Прогрессивная анкета (план №1) — снимает критику Галины «10 страниц холодной аудитории».
2. Апдейт питча (план №2) — Голикова + Конкуренты.
3. Кейсовая база Галины (план №6) — нужна для демо инвесторам, но требует №4 на месте, чтобы Галина грузила сторонние отчёты.

**Затем — продукт-экспансия (3–6 недель):**
4. Кросс-парсер ДНК (план №4).
5. Каталог опросников (план №3).
6. Модульные SKU (план №5).
7. Челленджи v0 (план №7).

**Финал — стратегические темы (6–12 недель):**
8. Multi-tenant «Семья» (план №8) — крупная переделка модели данных.
9. Прогнозный модуль (план №9).
10. B2B-партнёрский слой (план №10) — к встрече 04.06.
11. Контент-блок (план №11).
12. Документация-обновления (план №12) — параллельно по ходу.

## Принципы для всех планов

- **TDD по умолчанию** (Vitest уже в проекте после `2026-04-05`).
- **Малые коммиты** — после каждой задачи план содержит коммит.
- **Не плодить файлы** — следовать `src/lib/...` + `src/components/...` структуре.
- **Никаких backwards-compat shims** — проект на стадии MVP, ломать LocalStorage-форматы можно, добавив migration в `storage.ts`.
- **Документация — после задачи, не раньше:** `agent_docs/development-history.md` пополняется по факту, а не авансом.
