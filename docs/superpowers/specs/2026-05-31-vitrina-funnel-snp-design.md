# Дизайн: пересборка коммерческого слоя «Красивое долголетие» (витрина + воронка + 225 SNP)

- **Дата:** 2026-05-31
- **Источник требований:** Галина Хусаинова (эксперт-разработчик), чат GenesisBio, 27–29.05.2026 — 5 DOCX с составами блоков, голосовое 8:41 (29.05), текстовые правки 28–29.05.
- **Аватар/контекст:** женщина 35–60, см. ADR-001/002.

## Контекст и проблема

Текущая витрина `/products` (запущена 28.05, см. development-history) продаёт 5 генетических блоков как отдельные товары («наборчики генов»). Галина квалифицирует это как **принципиальную ошибку модели продаж** и задаёт другую рамку:

> Продаём **полный продукт «Красивое долголетие»**. Блоки — это **пошаговая воронка** (декомпозиция полного отчёта), способ зайти, а не самостоятельные товары.

Параллельно Галина закрыла главный «open question» проекта (`agent_docs/genetics-blocks/blocks-architecture-v1.md`, строки 66–71): прислала **финальные авторские rs-списки по всем 5 блокам**, сумма уникальных = 225 ген-точек.

## Цели

1. Перестроить `/products` под модель «полный продукт первым + пошаговая воронка с фиксированным порядком».
2. Завести источник истины для 225 SNP по 5 блокам.
3. Перенести дословный контент блоков (боли, вопросы, научные задачи) из DOCX в страницы блоков.
4. Внести правки копирайта и цен на `/early-access`.
5. Обеспечить научную/юридическую корректность формулировок (сайт = оферта).

## Зафиксированные решения (бизнес)

| Вопрос | Решение |

| Подарок Блок 4 | При **любых 2 платных блоках** из {1,2,3,5} в корзине без пакета — Блок 4 «Мозг/сон/стресс» добавляется строкой 0 ₽ |

## Архитектура решения

Стек: Next.js 16 (App Router) + Drizzle ORM + PostgreSQL + Zustand (cart) + shadcn/ui + Vitest. Витрина DB-driven (`tests` table, `src/lib/products/blocks.ts`).

### A. Источник истины SNP — `src/lib/genetics/blocks-snp.ts`
- Новый типизированный модуль: массив 5 блоков, каждый — `{ slug, title, price, snps: { gene: string; rs: string }[] }` дословно из DOCX.
- Почему модуль, а не БД: данные научные/статичные, ревьюятся в PR, переиспользуются в 3 местах (карточки блоков, подсветка ген-точек в ЛК после загрузки ДНК-отчёта, будущее расширение квиза на блоки 2–5).
- БД-поле `tests.markersCount` — производное (синхронизируется миграцией из модуля).
- **Инвариант (тест):** число уникальных `gene:rs` по всем блокам = 225; новые/пересечения совпадают с DOCX (Б2: 22/6, Б3: 15, Б4: 9, Б5: 128).
- Исходники для переноса: `downloads/telegram_-5020520887_607772..607776_*.docx` (архивируются в `agent_docs/genetics-blocks/source-galina-2026-05/`).

### B. Витрина `/products` — `src/app/(public)/products/page.tsx`
- **Герой:** полный пакет «Красивое долголетие» 65 000 (все 5 блоков + 2 ч консультации; якорь 89 900, выгода 24 900).
- **Блок воронки:** пошаговый путь — Блок 1 «Питание» как вход (15 900) → 2 → 3, Трио Wellness (27 900), подарок Блок 4, Блок 5 отдельным «самостоятельным» треком.
- Удалить карточку пакета «Пошаговая сборка 75 000».
- Терминологическая зачистка по всей странице: убрать «наборчики/поштучная сборка» → «поэтапная/пошаговая»; «N генов» → «N ген-точек (SNP)».
- `calculateAnchorPrice/Savings` в `src/lib/products/blocks.ts` — сверить с новой тарификацией.

### C. Логика корзины — `src/stores/cart-store.ts` + тесты
- **Гейтинг:** `addItem(block)` для Блока 2/3/4 запрещён, если в корзине нет Блока 1 и он не куплен. Блок 5 и пакеты — без ограничений. UX при отказе: тост «Начните с Блока 1 «Питание»» + действие «Добавить Блок 1».
- **Подарок:** дериватор — при ≥2 платных блоков из {1,2,3,5} и отсутствии пакета добавить Блок 4 строкой 0 ₽ (`isGift: true`); при падении ниже 2 — снять. В пакетах не применяется.
- Логика выносится в чистую тестируемую функцию (редьюсер состояния корзины), Zustand-store вызывает её.
- **Тесты (Vitest):** гейтинг (отказ без Б1; разрешение с Б1; Б5 свободно), подарок (появление при 2 блоках, снятие при <2, неприменение к пакетам), пересчёт суммы.

### D. Контент страниц блоков — `src/app/(public)/products/[slug]/page.tsx`
- Дословный перенос из DOCX (без перефраза ИИ — требование Галины): главные боли, таблица «как говорит клиент → что помогает понять → ген-точки», научные задачи, перечень/счётчик ген-точек.
- Блоки 2–5 получают контент впервые; Блок 1 — сверить с присланным DOCX.
- Источник контента и счётчиков — модуль из (A) + текстовые блоки из DOCX.

### E. Лендинг `/early-access` — `src/app/(public)/early-access/page.tsx`
Правки 28–29.05:
- «От «что со мной не так» — к персональному протоколу» → **«Шаг за шагом к персональному протоколу»**.
- Убрать боль «…которая хочет оставаться собой».
- «база за 14 900» → корректно: Блок 1 = 15 900 (база = Блок 1 «Питание»).
- Переписать «Мозг, сон, стресс и мотивация — в подарок» (нельзя «мозги в подарок»).
- Убрать «Почему Genesis, а не другая лаборатория» (Genesis — не лаборатория).
- Полный пакет → **65 000** (55 900 удалить). Трио Wellness 27 900 — оставить.
- «Понимание, какие анализы и привычки требуют большего внимания именно у вас» — оставить как есть (подтверждено Галиной).

- Тест-инвариант 225 SNP зелёный; тесты гейтинга/подарка зелёные.
- `/products`: герой = полный пакет 65 000; пакет 75 000 отсутствует; терминология вычищена.

- Цены консистентны между `/products`, `/early-access` и БД.
