## Контекст и цели

Модуль реализует слой **Onboarding & Questionnaire** из архитектуры Genesis (`agent_docs/architecture.md`) — стартовое анкетирование (анамнез) как одна из трёх воронок входа. Пользователь проходит биохакинг-анкету (~100 полей, 9 шагов, ~25 минут) и получает структурированную сводку с автоматическими подсветками рисков.

**Роль в продукте:**
- **Лид-воронка** — публичный доступ без регистрации, конверсия на выходе
- **Инструмент для пользователей** — сохранение анкеты в профиле для будущей интеграции с генетикой, анализами крови и биовозрастом
- **Источник контекста для Genesis Coach** — структурированный анамнез как основа для персонализированного протокола (будущее)

## Принятые решения

| # | Вопрос | Решение |
|---|---|---|
| 1 | Назначение/аудитория | **Гибрид**: анонимно заполняется, но для сохранения истории и интерпретации нужна регистрация |
| 2 | Выход | **Сводка + подсветки + интеграция с профилем** (место для будущих данных) |
| 3 | UX-структура | **Многошаговый визард** с прогресс-баром, 9 шагов |
| 4 | Сохранение прогресса | **LocalStorage** для анонимов + **soft gate** на регистрацию в сводке |
| 5 | Подсветки в MVP | **15 детерминированных правил-триггеров** с дисклеймерами |

### Модель данных

**Новая таблица `questionnaire_sessions`** в `src/db/schema.ts`:

```ts
questionnaireSessions: {
  id: uuid (pk)
  userId: uuid | null        // null = анонимный
  sessionToken: text         // для анонимов, хранится в LocalStorage
  status: 'draft' | 'completed'
  currentStep: integer (0..9)
  answers: jsonb             // { step1: {...}, step2: {...}, ... }
  highlights: jsonb | null   // [{ruleId, severity, title, ...}]
  completedAt: timestamp | null
  email: text | null         // захваченный на soft-gate
  createdAt, updatedAt: timestamp
}
```

**Индексы:** `userId`, `sessionToken`, `status`.

**Стратегия:** JSONB-blob для MVP. Анкета будет эволюционировать, жёсткая нормализация 100+ полей в отдельные таблицы — преждевременная оптимизация. Нормализуем позже, когда структура стабилизируется и понадобится аналитика.

**Zod-схемы** (`src/lib/questionnaire/types.ts`) — строгая типизация каждого шага. Один источник правды для формы, валидации и интерпретатора правил.

### Поток данных

```
Анонимный пользователь:
  LocalStorage ("genesis_q_v1") → Последний шаг →
  POST /api/questionnaire/submit →
  questionnaireSessions (sessionToken, userId=null, status=completed) →
  Сводка + CTA регистрации

Авторизованный пользователь:
  После каждого шага: PATCH /api/questionnaire/session →
  questionnaireSessions (userId=..., status=draft) →
  Завершение: status=completed, highlights вычисляются серверно →
  Доступно в /dashboard/questionnaire

Привязка анонима к аккаунту:
  При регистрации после анонимного заполнения:
  POST /api/questionnaire/attach { sessionToken } →
  UPDATE set userId=... WHERE sessionToken=... AND userId IS NULL
```

### Движок подсветок

**Модуль:** `src/lib/questionnaire/highlights.ts`

Чистая детерминированная функция: `computeHighlights(answers) → Highlight[]`. Без LLM, без сайд-эффектов. Легко тестируется.

**Вычисление серверное** (в `/api/questionnaire/submit`) — один источник правды, клиент не может подделать результат.

**Формат правила:**

```ts
type Rule = {
  id: string
  category: 'nutrition' | 'medication' | 'lifestyle' | 'systems' | 'stress'
  severity: 'info' | 'attention' | 'warning'
  title: string
  description: string
  triggers: string[]         // какие ответы сработали (прозрачность)
  recommendation: string
  disclaimer: string         // обязательный
}
```

**15 правил MVP:**

| # | Категория | Правило | Триггер |
|---|---|---|---|
| 1 | nutrition | Возможный дефицит магния | судороги + тремор + нарушения сна + раздражительность (3+/4) |
| 2 | nutrition | Возможный дефицит B12/фолатов | утомляемость + одышка + бледность + жжение языка + туман (3+/5) |
| 3 | nutrition | Возможный дефицит железа | бледность + утомляемость + одышка + выпадение волос + пристрастие к непищевому (3+/5) |
| 4 | nutrition | Возможный дефицит витамина D | боли в костях + мышечная слабость + частые простуды + плохое зрение в сумерках (2+/4) |
| 5 | nutrition | Возможный дефицит Омега-3 | сухая кожа + выпадение волос + ломкость ногтей + сухость глаз + отсутствие жирной рыбы (3+/5) |
| 6 | nutrition | Возможный дефицит цинка | снижение вкуса/обоняния + медленное заживление + частые простуды + выпадение волос (2+/4) |
| 7 | nutrition | Низкое потребление клетчатки | <250 г овощей + «редко» крупы + склонность к запорам |

Каждая подсветка — карточка с цветовой индикацией (info/attention/warning), раскрываемыми триггерами и обязательным дисклеймером: **«Не является медицинским диагнозом. Рекомендуем обсудить со специалистом»**.

## UX-паттерны

**Прогресс-бар:** «Шаг 4 из 9 • ~16 мин» + визуальная полоска, sticky top на мобильных.

**Таблица частот** (переиспользуется для лекарств, БАД, продуктов):
- Десктоп: таблица с radio-кнопками «Никогда / Редко / Иногда / Часто»
- Мобильный: карточки с segmented control

**Чекбоксы симптомов** (30 штук): сетка 2 колонки на десктопе, 1 на мобильных, визуальный счётчик выбранных.

**Сохранение и выход:** кнопка «Сохранить и выйти» сверху справа → модалка «Прогресс сохранён. Вернитесь по ссылке `/questionnaire/start`».

**Сводка (Шаг 9 / страница result):**
- Верх: блок подсветок (15 правил), сортировка warning → attention → info
- Середина: аккордеон «Ваши ответы» по всем секциям
- Плавающий CTA: «Зарегистрируйтесь, чтобы сохранить анкету и подключить анализы» (для анонимов)
- Кнопка «Отправить на email»

## Технический подход

**Стек** (используем существующий):
- Next.js App Router
- react-hook-form + Zod (паттерн из `src/components/calculator/calculator-form.tsx`)
- Drizzle ORM + PostgreSQL
- shadcn/ui (Card, Input, Label, Button, Select, Dialog, Accordion, **+ новые: RadioGroup, Checkbox, Progress**)

**State management:**
- Per-step form: `react-hook-form` с Zod-резолвером на каждом шаге
- Wizard state: локальный `useState` в `wizard.tsx`, хранит `currentStep` и аккумулированные `answers`
- Persistence: после каждого шага — `storage.save()` → LocalStorage (анонимы) или `PATCH /api/questionnaire/session` (авторизованные)

**API-эндпоинты:**
- `POST /api/questionnaire/submit` — создать/завершить сессию, вычисляет highlights серверно
- `GET /api/questionnaire/session?token=...` — получить черновик по токену
- `PATCH /api/questionnaire/session` — обновить текущий шаг/ответы
- `POST /api/questionnaire/attach` — привязать анон-сессию к userId

**Безопасность:**
- Rate-limiting на `/api/questionnaire/submit` (5/min/IP)
- `sessionToken` — криптостойкий random (nanoid, 21 символ)
- Проверка `userId === session.userId` на PATCH/GET
- Sanitize email перед сохранением

## Тестирование

- **Unit:** каждое из 15 правил в `highlights.ts` с фикстурами (input → expected highlights)
- **Unit:** Zod-схемы каждого шага (валидные/невалидные кейсы)
- **Integration:** API-эндпоинты (submit с невалидными данными, attach с несуществующей сессией)

## Решения по граничным вопросам

- **Email-отправка сводки** — **в MVP кнопка-заглушка** (show toast «Функция скоро»). Реальную отправку добавим отдельной задачей, когда выберем провайдер (Resend/SES). Email всё равно захватывается на soft-gate.
- **PDF-экспорт сводки** — **не в MVP**, добавим итерацией после сбора фидбека.
- **Шаринг по ссылке** — публичный URL `/questionnaire/result/[token]` доступен без пароля; `sessionToken` (nanoid 21 симв.) обеспечивает достаточную защиту от перебора.

## Что НЕ входит в MVP

- LLM-интерпретация (только детерминированные правила)
- PDF-экспорт
- Интеграция с импортом анализов (появится в следующих фазах)
- Многоязычность (только русский)
- Встроенный messaging со специалистом
- История изменений анкеты (только последняя версия)

## Связь с архитектурой Genesis

- **Слой:** Onboarding & Questionnaire (слой 0 в `agent_docs/architecture.md`)
- **Воронка:** одна из трёх точек входа (наряду с генетикой и анализами крови)
- **Продолжение:** результаты анкеты → контекст для Genesis Coach → подсветки в объединённой картине с генетикой и биомаркерами (фазы M7-12)
